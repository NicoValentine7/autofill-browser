いろんなフォームのクセを一気に叩ける検証ページやで。

単純な問い合わせフォームだけやなくて、英語 checkout、郵便番号や電話の分割入力、属性が弱い legacy フォーム、誤爆しやすい username や cardholder name までまとめた。拡張の dev 版を reload したあと、このページで実戦っぽく回す想定や。

日本語の基本フォーム

氏名、メール、電話、会社名、都道府県セレクト、住所1/2 の基本パターンや。

name email tel organization postal-code address-level1 address-line1 address-line2

分離された住所フォーム

DeepL みたいに `市町村` と `住所欄1` と `住所欄2` が分かれてるケースや。`住所欄1` は街区だけ、`住所欄2` は建物名だけが理想やで。

英語 checkout パターン

`Family name / Given name / Street address / Region` みたいな英語ラベルでも拾えるかを見るやつや。

分割郵便番号・分割電話

昔ながらのフォームやモバイル UI でよくある、複数 input に分かれたパターンや。

`zip1 / zip2`、`tel1 / tel2 / tel3`、`前半 / 後半 / 末尾` みたいな token を読む想定や。

legacy フォーム

autocomplete が無かったり、placeholder や aria-label しかヒントが無いフォームや。ここで拾えんと実サイトでしんどい。

誤爆トラップ

`name` を含んでても埋めたくない欄をまとめた。ここに値が入るなら matcher の除外がまだ甘いってことや。

動的フォーム / モーダル風

ボタンであとからフォームを差し込む。MutationObserver と SPA っぽい追加 UI をまとめて確認できるやつや。
